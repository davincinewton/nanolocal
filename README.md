## ğŸš€ Quick Start

> [!TIP]

**1. Initialize**

```json

      "apiKey": "sk-or-v1-xxx"
    }
  },
  "agents": {
    "defaults": {
      "model": "anthropic/claude-opus-4-5"
    }
  }
}
```

**With SearXNG (Free Web Search):**
```json
{
  "providers": {
    "openrouter": {
      "apiKey": "sk-or-v1-xxx"
    }
  },
  "agents": {
    "defaults": {
      "model": "anthropic/claude-opus-4-5"
    }
  },
  "tools": {
    "web": {
      "search": {
        "searxng_url": "http://localhost:8080",
        "max_results": 5
      }
    }
  }
}
```

> [!TIP]
> SearXNG provides free, self-hosted web search. Deploy with `docker run -p 8080:8080 searxng/searxng:latest` or use a public instance.

**3. Chat**

```bash
nanobot agent -m "What is 2+2?"
```

That's it! You have a working AI assistant in 2 minutes.

## ğŸ–¥ï¸ Advanced Local Model Support

nanobot supports ANY OpenAI-compatible local server, not just vLLM. Configure custom endpoints, authentication, and model parameters for complete local model control.

### Example Configurations

**vLLM Server:**
```json
{
  "providers": {
    "vllm": {
      "apiKey": "dummy",
      "apiBase": "http://localhost:8000/v1"
    }
  },
  "agents": {
    "defaults": {
      "model": "meta-llama/Llama-3.1-8B-Instruct"
    }
  }
}
```

**Custom OpenAI-Compatible Server:**
```json
{
  "providers": {
    "custom_local": {
      "apiKey": "your-custom-key",
      "apiBase": "http://localhost:1234/v1",
      "extraHeaders": {
        "Authorization": "Bearer your-token"
      }
    }
  },
  "agents": {
    "defaults": {
      "model": "custom-model-name"
    }
  }
}
```

**Ollama Integration:**
```json
{
  "providers": {
    "llamacpp": {
      "apiKey": "dummy",
      "apiBase": "http://localhost:11434/v1"
    }
  },
  "agents": {
    "defaults": {
      "model": "llama3.2:latest"
    }
  }
}
```

### Usage

**1. Start your local server**
```bash
# vLLM example
vllm serve meta-llama/Llama-3.1-8B-Instruct --port 8000

# Ollama example  
llama-cpp serve

# Custom server
your-local-server --port 1234
```

**2. Configure** (`~/.nanobot/config.json`)
```json
{
  "providers": {
    "local_llm": {
      "apiKey": "dummy",
      "apiBase": "http://localhost:8000/v1",
      "timeout": 300,
      "maxRetries": 3
    }
  }
}
```

**3. Chat**

```bash
nanobot agent -m "Hello from my local LLM!"
```

> [!TIP]
> Local servers can use any non-empty `apiKey`. Configure timeout, retries, and custom headers for advanced scenarios.

### Security

> [!TIP]
> For production deployments, set `"restrictToWorkspace": true` in your config to sandbox the agent.

| Option | Default | Description |
|--------|---------|-------------|
| `tools.restrictToWorkspace` | `false` | When `true`, restricts **all** agent tools (shell, file read/write/edit, list) to the workspace directory. Prevents path traversal and out-of-scope access. |
| `channels.*.allowFrom` | `[]` (allow all) | Whitelist of user IDs. Empty = allow everyone; non-empty = only listed users can interact. |

> [!TIP]
> The `-v ~/.nanobot:/root/.nanobot` flag mounts your local config directory into the container, so your config and workspace persist across container restarts.

Build and run nanobot in a container:

```bash
# Build the image
docker build -t nanobot .

# Initialize config (first time only)
docker run -v ~/.nanobot:/root/.nanobot --rm nanobot onboard

# Edit config on host to add API keys
vim ~/.nanobot/config.json

# Run gateway (connects to Telegram/WhatsApp)
docker run -v ~/.nanobot:/root/.nanobot -p 18790:18790 nanobot gateway

# Or run a single command
docker run -v ~/.nanobot:/root/.nanobot --rm nanobot agent -m "Hello!"
docker run -v ~/.nanobot:/root/.nanobot --rm nanobot status
```
